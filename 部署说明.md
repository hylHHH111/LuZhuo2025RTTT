# 留言板部署说明

## 文件说明

- `worker.js` - Cloudflare Worker 后端脚本
- `home.html` - 已添加留言板功能的首页
- `部署说明.md` - 本说明文件

---

## Cloudflare 部署步骤

### 第一步：创建 KV 命名空间

1. 登录 [Cloudflare 控制台](https://dash.cloudflare.com)
2. 左侧菜单点击 **"Workers & Pages"**
3. 点击 **"KV"** 标签
4. 点击 **"创建命名空间"**
5. 名称填写：`guestbook`
6. 点击 **"添加"**

---

### 第二步：创建 Worker

1. 在 Workers & Pages 页面，点击 **"创建"**
2. 选择 **"Worker"**
3. 输入 Worker 名称（例如：`luzhou-guestbook`）
4. 点击 **"部署"**
5. 部署完成后，点击 **"编辑代码"**

---

### 第三步：绑定 KV 到 Worker

1. 在 Worker 编辑页面，点击 **"设置"** 标签
2. 左侧选择 **"变量"**
3. 找到 **"KV 命名空间绑定"** 部分
4. 点击 **"添加绑定"**
5. 填写：
   - 变量名称：`GUESTBOOK_KV`
   - KV 命名空间：选择刚才创建的 `guestbook`
6. 点击 **"保存"**

---

### 第四步：上传 Worker 代码

1. 回到 **"代码"** 标签
2. 删除默认代码，将 `worker.js` 的全部内容粘贴进去
3. 点击 **"保存并部署"**

---

### 第五步：获取 Worker URL

1. 部署成功后，在 Worker 页面顶部可以看到 URL
2. 格式类似：`https://luzhou-guestbook.xxxxx.workers.dev`
3. 复制这个 URL

---

### 第六步：修改前端代码

1. 打开 `home.html`
2. 找到第 882 行（或搜索 `WORKER_URL`）
3. 将：
   ```javascript
   const WORKER_URL = 'https://your-worker.your-subdomain.workers.dev';
   ```
   修改为实际的 Worker URL，例如：
   ```javascript
   const WORKER_URL = 'https://luzhou-guestbook.xxxxx.workers.dev';
   ```
4. 保存文件

---

### 第七步：部署网站

1. 将修改后的 `home.html` 提交到 GitHub
2. Cloudflare Pages 会自动重新部署
3. 等待部署完成后即可使用留言板

## 管理功能说明

### 管理密钥

- 默认管理密钥：`luzhou2025`
- 可以在 `worker.js` 第 13 行修改
- 修改后需要重新部署 Worker
- 同时需要在 `home.html` 第 885 行同步修改

### 删除留言步骤

1. 在留言板页面点击 **"管理模式"** 按钮
2. 输入管理密钥
3. 点击 **"确认"**
4. 每条留言右上角会出现 **"删除"** 按钮
5. 点击即可删除对应留言

✅ 实时显示留言（最新在前）  
✅ 最多显示 222 条留言（超过自动覆盖旧的）  
✅ 可选输入昵称（不填显示"匿名用户"）  
✅ 显示发布时间  
✅ 前后端双重敏感词过滤  
✅ 管理员删除功能  
✅ 完全免费（使用 Cloudflare 免费额度）

## 注意事项

1. **首次部署后**，留言板可能显示"加载失败"，这是因为还没有留言数据，发布第一条留言后即可正常显示
2. **管理密钥**请妥善保管，不要泄露
3. **免费额度**：Cloudflare Workers 每天 10 万次请求，足够个人网站使用
4. **数据存储**：留言数据存储在 Cloudflare KV 中，理论上是永久的

## 故障排查

### 留言发布失败

1. 检查 Worker URL 是否正确配置
2. 检查 Worker 是否正常运行
3. 检查 KV 绑定是否正确

### 无法加载留言

1. 检查浏览器控制台是否有报错
2. 检查 Worker URL 是否可以访问
3. 检查 CORS 配置是否正确

### 管理功能无法使用

1. 确认管理密钥前后端一致
2. 检查 Worker 代码中的 ADMIN_KEY
3. 检查前端代码中的 ADMIN_KEY
